# 投资管理系统数据交互说明

## 1. 数据库与表结构

目前系统使用的是 PostgreSQL 数据库，通过 Prisma ORM 进行管理，总共创建了 5 张核心数据表。

### 1.1 DashboardOverview (数据大屏总览)
存储数据大屏顶部的四个核心卡片的数据：规模总览、收益核心、效率核心、风险总览。
```sql
CREATE TABLE "DashboardOverview" (
    "id" TEXT NOT NULL,
    "totalProjectCount" INTEGER NOT NULL DEFAULT 0,
    "totalInvestment" DOUBLE PRECISION,
    "fundCount" INTEGER NOT NULL DEFAULT 0,
    "newProjectCount" INTEGER NOT NULL DEFAULT 0,
    "irrMedian" DOUBLE PRECISION,
    "dpiDistribution" VARCHAR(50),
    "avgReturnMultiple" DOUBLE PRECISION,
    "exitWinRate" DOUBLE PRECISION,
    "avgProjectDuration" INTEGER,
    "invalidEfficiency" DOUBLE PRECISION,
    "approvalPassRate" DOUBLE PRECISION,
    "highRiskProjectCount" INTEGER NOT NULL DEFAULT 0,
    "compliancePendingCount" INTEGER NOT NULL DEFAULT 0,
    "todayMeetingCount" INTEGER NOT NULL DEFAULT 0,
    "updatedAt" TIMESTAMP(3) NOT NULL,
    CONSTRAINT "DashboardOverview_pkey" PRIMARY KEY ("id")
);
```

### 1.2 DashboardChart (数据大屏图表)
存储赛道分布（饼图）、流程趋势（柱状图）、行业对标（折线图）和人员效能排行的数据。为了整合展示，不同图表的数据通过对应的字段标识进行存储.
```sql
CREATE TABLE "DashboardChart" (
    "id" TEXT NOT NULL,
    "trackName" VARCHAR(50),
    "trackInvestAmount" DOUBLE PRECISION,
    "trackRatio" DOUBLE PRECISION,
    "stageName" VARCHAR(50),
    "stageProjectCount" INTEGER,
    "statisticMonth" VARCHAR(20),
    "quarter" VARCHAR(20),
    "fundIrr" DOUBLE PRECISION,
    "industryTopIrr" DOUBLE PRECISION,
    "industryAvgIrr" DOUBLE PRECISION,
    "managerName" VARCHAR(50),
    "managerProjectCount" INTEGER,
    "managerAvgCycle" INTEGER,
    "managerEfficiencyScore" INTEGER,
    CONSTRAINT "DashboardChart_pkey" PRIMARY KEY ("id")
);
```

### 1.3 DashboardTodo (数据大屏待办事项)
存储数据大屏下方的我的待办决策事项列表数据。
```sql
CREATE TABLE "DashboardTodo" (
    "id" TEXT NOT NULL,
    "type" VARCHAR(50),
    "title" VARCHAR(255),
    "projectName" VARCHAR(255),
    "submitter" VARCHAR(50),
    "submitTime" TIMESTAMP(3),
    "deadline" TIMESTAMP(3),
    "priority" VARCHAR(20),
    "operation" VARCHAR(20),
    "createdAt" TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT "DashboardTodo_pkey" PRIMARY KEY ("id")
);
```

### 1.4 Project (项目列表)
存储项目管理模块中的实体项目列表。
```sql
CREATE TABLE "Project" (
    "id" TEXT NOT NULL,
    "name" VARCHAR(255) NOT NULL,
    "logo" VARCHAR(20),
    "description" TEXT,
    "stage" VARCHAR(50) NOT NULL,
    "tags" VARCHAR(50) NOT NULL,
    "round" VARCHAR(50),
    "managerId" VARCHAR(100),
    "managerName" VARCHAR(100),
    "totalInvestment" DOUBLE PRECISION,
    "irr" DOUBLE PRECISION,
    "createdAt" TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP,
    "updatedAt" TIMESTAMP(3) NOT NULL,
    CONSTRAINT "Project_pkey" PRIMARY KEY ("id")
);
```

### 1.5 Strategy (策略列表)
存储策略看板中的策略分析记录。
```sql
CREATE TABLE "Strategy" (
    "id" TEXT NOT NULL,
    "iconName" VARCHAR(50),
    "name" VARCHAR(255) NOT NULL,
    "frameworkName" VARCHAR(255),
    "description" TEXT,
    "managerName" VARCHAR(100),
    "projectCount" INTEGER NOT NULL DEFAULT 0,
    "totalInvestment" VARCHAR(50),
    "returnRate" VARCHAR(50),
    "createdAt" TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP,
    "updatedAt" TIMESTAMP(3) NOT NULL,
    CONSTRAINT "Strategy_pkey" PRIMARY KEY ("id")
);
```

## 2. 后端接口实现

系统采用了 Next.js 的 **App Router Route Handlers** (API路由) 来实现 RESTful 接口。

1. **lib/prisma.ts**
   - 创建了 Prisma Client 单例，用于全局安全地复用数据库连接，避免开发环境下的连接数耗尽问题。
2. **app/api/dashboard/route.ts**
   - 处理 `/api/dashboard` (GET请求)
   - 逻辑：通过 `Promise.all` 并发向 Prisma 请求：
     - `DashboardOverview.findFirst()`
     - `DashboardChart.findMany()`
     - `DashboardTodo.findMany()`
   - 将这三部分组合为一个 JSON 对象返回给前端。
3. **app/api/projects/route.ts**
   - 处理 `/api/projects` (GET请求)
   - 逻辑：通过 `prisma.project.findMany()` 取出所有项目列表并倒序返回。
4. **app/api/strategies/route.ts**
   - 处理 `/api/strategies` (GET请求)
   - 逻辑：通过 `prisma.strategy.findMany()` 取出所有策略模块并倒序返回。

## 3. 前端页面渲染

前端已对接好数据库，所有静态/写死的 JSON 模拟变量已经被完全删除，全部改为请求 API 的方式并动态渲染。

### 修改的文件与逻辑：
1. **components/pages/dashboard.tsx**
   - 移除所有的静态配置变量 (`trackDistribution` 等模拟常量)。
   - **渲染逻辑**：在组件内使用 `useEffect` 调用 `fetch('/api/dashboard')` 取得后端数据存于 `dbData` 的状态中。
   - 然后通过安全链式调用 (`dbData?.overview?.xxx ?? 0`) 将接口字段动态解析并注入到 Recharts 和 DOM 元素中。
