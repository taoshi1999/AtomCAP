# 投资项目管理系统 - 数据库实现文档

## 一、概述

本系统实现了投资项目的数据库管理功能，包括PostgreSQL数据库配置、Prisma ORM集成、以及前后端数据交互。

## 二、技术栈

| 组件 | 技术 |
|------|------|
| 数据库 | PostgreSQL 18 |
| ORM | Prisma 5.22 |
| 后端框架 | Next.js API Routes |
| 前端框架 | Next.js (React 19) |
| 连接驱动 | pg (node-postgres) |

## 四、数据库表结构

### 4.1 projects 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT | 主键 (Prisma CUID) |
| code | TEXT | 项目编号 (唯一) |
| name | TEXT | 公司名称 |
| logo | TEXT | 项目Logo |
| description | TEXT | 项目描述 |
| tags | TEXT[] | 赛道标签数组 |
| status | TEXT | 项目状态 (默认"Pending") |
| valuation | TEXT | 估值 |
| round | TEXT | 投资轮次 |
| ownerId | TEXT | 负责人ID |
| ownerName | TEXT | 负责人姓名 |
| strategyId | TEXT | 关联策略ID |
| strategyName | TEXT | 关联策略名称 |
| createdAt | TIMESTAMP | 创建时间 |
| updatedAt | TIMESTAMP | 更新时间 |

### 4.2 索引

```sql
CREATE INDEX projects_code_idx ON projects(code);
CREATE INDEX projects_status_idx ON projects(status);
CREATE INDEX projects_ownerId_idx ON projects("ownerId");
CREATE INDEX projects_strategyId_idx ON projects("strategyId");
CREATE INDEX projects_createdAt_idx ON projects("createdAt");
```

**注意**: PostgreSQL 中 camelCase 字段名需要用双引号包裹，如 `"ownerId"`。

## 五、API接口

### 5.1 获取项目列表

```
GET /api/projects
```

查询参数:
- `status` - 按状态筛选
- `ownerId` - 按负责人筛选
- `strategyId` - 按策略筛选

响应示例:
```json
[
  {
    "id": "clx1234567890",
    "code": "PRJ2604100001",
    "name": "Moonshot AI",
    "logo": "M",
    "description": "New generation AI search and dialogue platform",
    "tags": ["AI", "Series A"],
    "status": "Mid-stage",
    "valuation": "1B USD",
    "round": "Series A",
    "ownerId": "lisi",
    "ownerName": "Li Si",
    "strategyId": "2",
    "strategyName": "LLM Applications",
    "createdAt": "2023-09-20T00:00:00.000Z"
  }
]
```

### 5.2 创建项目

```
POST /api/projects
```

请求体:
```json
{
  "name": "Project Name",
  "logo": "P",
  "description": "Project description",
  "tags": ["AI", "Series A"],
  "valuation": "1B USD",
  "round": "Series A",
  "ownerId": "zhangwei",
  "ownerName": "Zhang Wei",
  "strategyId": "2",
  "strategyName": "LLM Applications"
}
```

注意: `code` 和 `status` 字段自动生成

响应: 201 Created

### 5.3 初始化数据

```
POST /api/projects/init
```

将前端示例数据写入数据库（仅在数据库为空时执行）

## 六、项目编号规则

格式: `PRJ` + `YYMMDD` + `XXXX` (4位随机数)

示例: `PRJ2604101234`

### 7.2 项目创建流程

1. 用户在ProjectsGrid组件中填写项目信息
2. 点击"创建"按钮，生成PendingProject对象
3. 进入变更审批流程
4. 审批通过后，调用 `POST /api/projects` 保存到数据库

## 九、启动步骤

### 9.1 数据库准备

```bash
# 创建数据库
psql -U postgres -c "CREATE DATABASE atomcap;"

# 创建表
psql -U postgres -d atomcap -f prisma/migrations/create_table.sql

# 导入示例数据
psql -U postgres -d atomcap -f prisma/migrations/seed_data.sql
```

**Windows 编码问题**: 如果遇到编码错误，SQL 文件已使用英文内容避免编码问题。

### 9.2 启动应用

```bash
pnpm install
pnpm exec prisma generate
pnpm run dev
```

### 9.3 初始化数据（可选）

```bash
curl -X POST http://localhost:3000/api/projects/init
```
